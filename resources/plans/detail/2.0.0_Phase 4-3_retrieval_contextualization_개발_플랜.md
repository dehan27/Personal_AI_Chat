# 2.0.0 Phase 4-3 — Retrieval Contextualization 개발 플랜

## Context

Phase 4-2 머지 직후 실사용 중 발견된 회귀 상황:

```
user: 경조사 규정 알려줘
bot : (경조사 규정 전체 답변 — 본인 상 500만원, 배우자 상 100만원, ...)
user: 비싼거
bot : 외주용역 5억 이상 turnkey 결재 라인 ...   ❌ 엉뚱한 답
```

원인은 라우터가 아니라 `chat/services/single_shot/pipeline.py:35-38` 의 구조 자체다:

```python
chunk_hits = retrieve_documents(question)   # ← history 안 씀
qa_hits    = find_canonical_qa(question)    # ← 여기도 안 씀
...
messages   = build_single_shot_messages(question, chunk_hits, qa_hits, history)  # LLM 단계에만 history
```

**retrieval 은 single-turn, LLM 은 multi-turn** 이 어긋나 있음. 후속 질문 "비싼거" 가 `search_chunks("비싼거", ...)` 에 그대로 들어가 "비싼" 이 포함된 전혀 다른 문서(5억 외주용역 조항)가 상위로 잡히고, LLM 은 제공된 자료 안에서만 답을 뽑기 때문에 맥락이 있는데도 잘못된 답변을 낸다.

Phase 4-3 의 목표는 **retrieval 전에 "self-contained 검색어" 를 만들어 주는 쿼리 재작성 단계를 삽입** 해 이 회귀를 해소하는 것. LLM 최종 응답 단계는 건드리지 않는다 (원본 질문 그대로 보여주고 history 로 맥락 제공 — 기존 방식 유지).

## 핵심 설계 결정

### 1. 재작성 단계 위치

`pipeline.py` 에 단 한 줄 삽입:

```python
def run_single_shot(question, history=None):
    history = history or []

    # 신규: retrieval 전에 self-contained 검색어 생성
    search_query = rewrite_query_with_history(question, history)

    chunk_hits = retrieve_documents(search_query)
    qa_hits    = find_canonical_qa(search_query)
    ...
    # LLM 은 원본 question + history 그대로
    messages = build_single_shot_messages(question, chunk_hits, qa_hits, history)
```

장점:
- `search_chunks` / `find_canonical_qa` 시그니처 건드리지 않음 → ripple effect 최소
- 원본 `question` 은 그대로 LLM·UI·ChatLog 에 흘러감 → 사용자가 입력한 문구 보존
- workflow / agent 에서도 동일 helper 를 앞단에 끼우기만 하면 재사용 가능

### 2. `rewrite_query_with_history` 동작

```python
def rewrite_query_with_history(question: str, history: List[Dict]) -> str:
    # 1) history 가 비어있으면 원본 반환 (첫 질문)
    if not history:
        return question

    # 2) LLM 호출: "대화 맥락 + 현재 질문" → self-contained 한 줄
    try:
        rewritten = _call_rewriter_llm(question, history)
    except Exception as exc:
        logger.warning('쿼리 재작성 실패, 원본 사용: %s', exc)
        return question

    # 3) LLM 이 "변경 불필요" 를 의미하는 sentinel 출력 시 원본 반환
    if _is_noop(rewritten):
        return question

    logger.info('쿼리 재작성: %r → %r', question, rewritten)
    return rewritten
```

- **sentinel**: 프롬프트에서 "질문이 이미 자립적이면 정확히 `NOOP` 만 출력" 규약. 파싱 비용 없이 조기 종료.
- **실패 시 원본**: OpenAI 장애·타임아웃·파싱 오류에서 기존 동작 유지. 회귀 0 보장.
- **history 슬라이싱**: 전부 쓰지 않고 최근 user/assistant **3턴** 만 사용 (토큰 예산 + 오래된 맥락이 오히려 노이즈). 상수 `REWRITE_HISTORY_TURNS = 3` 로 모듈 상단.

### 3. 모델 선택

기존 `llm.py:16` 의 `OPENAI_MODEL` 환경변수(기본 `gpt-4o-mini`)를 **그대로 재사용**. 재작성은 가볍고 지연에 민감하므로 이미 cheap model 인 기본값이 적합. 별도 env var 도입하지 않음 (YAGNI).

LLM 호출 경로:
- 재작성용 messages 를 직접 구성해 `chat/services/single_shot/llm.py` 의 `run_chat_completion` 재사용.
- 재작성 호출도 `record_token_usage` 로 별도 TokenUsage 레코드 기록 (모델·토큰 모두 실제 값).

### 4. 프롬프트 외부화

`assets/prompts/query_rewriter.md` 로 분리하고 Phase 1 의 `prompt_registry` 에 등록 → BO Prompt 관리 페이지에서 운영자가 직접 수정 가능. 코드 재배포 없이 재작성 품질 튜닝 가능.

프롬프트 초안 (단일 user message 로 전달, system 은 짧은 지시):

```
이전 대화와 현재 질문이 주어집니다. 현재 질문이 이전 맥락에 의존하는
지시어·대명사(이거/저거/비싼거/첫번째 등)를 포함하거나 의미가 모호하면,
이전 대화를 반영해 **단독으로도 이해되는 검색어 한 줄** 로 바꿔주세요.

규칙:
- 한 줄, 특수기호 없이 자연어 한국어.
- 현재 질문이 이미 자립적이면 정확히 "NOOP" 만 출력.
- 불필요한 말(따옴표/설명/접두어) 금지.
```

### 5. 관측

- `logger.info('쿼리 재작성: %r → %r', question, rewritten)` 한 줄. 너무 잦으면 운영 로그 노이즈이므로 INFO 한 라인으로 제한.
- 실패는 `logger.warning` 로 구분.
- 후속 단계에서 Kibana / Grafana 붙일 때 이 로그만 파싱하면 재작성 품질 추적 가능.

### 6. QA 캐시 영향

재작성된 쿼리로 `find_canonical_qa` 를 호출하면 CanonicalQA 매칭 임베딩이 달라질 수 있음. 다만:
- CanonicalQA 매칭은 임베딩 **코사인 유사도 0.80 임계값** (`qa_retriever.py:19`). "경조사 중 가장 비싼 항목" 으로 재작성되면 오히려 정확한 매칭 증가 기대.
- 캐시 히트 조건은 top-1 유사도 ≥ 0.88 (`qa_retriever.py:28`). 재작성으로 유사도 분포가 흔들릴 수 있으므로 검증 단계에서 회귀 관찰.
- 문제가 있으면 Phase 4-3 후속에서 `find_canonical_qa` 에 원본+재작성 모두 조회 후 OR 결합 옵션 도입.

### 7. 테스트

`chat/tests.py` 는 비어있어서 Phase 4-3 에 단위 테스트 3~4 개를 얹는다:
- `test_rewriter_empty_history_returns_original` — history=[] 면 그대로 반환.
- `test_rewriter_noop_sentinel_returns_original` — LLM 이 `NOOP` 만 내면 원본.
- `test_rewriter_handles_llm_failure` — OpenAI 예외 mock → 원본 반환 (fallback).
- (선택) `test_rewriter_follow_up_integrates_prev_turn` — mock 으로 재작성 결과 검증.
