# 2.0.0 Phase 4-4 — OpenAI 사용량 위젯 개발 플랜

## Context

BO 대시보드는 현재 우리 서비스가 `TokenUsage` 테이블에 직접 쌓은 **내부 집계**만 보여준다. OpenAI 가 실제로 집계해 청구 기준으로 삼는 **조직 전체 사용량**은 우리 앱에서 확인할 길이 없었다.

Phase 4-4 는 대시보드 타이틀 우측에 버튼 하나를 추가해, 클릭하면 화면 중앙에 **모달 팝업**이 뜨고 그 안에서 **OpenAI Usage / Costs API** 로 실시간 집계를 받아 보여주도록 한다. 기존 카드·일별 표는 그대로 둔다(내부 기능별 관측 용도).

Admin 키(`sk-admin-...`) 는 일반 `OPENAI_API_KEY` 로는 접근 불가한 `/v1/organization/usage/*` 엔드포인트를 쓰기 위해 별도로 필요하며, 이미 운영자가 `.env` 에 `OPENAI_ADMIN_KEY` 로 저장해 두었다.

## 핵심 설계 결정

### 1. 두 개의 시간축

모달은 두 섹션으로 나뉜다.

- **전체 누적** — `start_time` 을 `2026-01-01 00:00 UTC` 로 고정, `bucket_width=1d` 로 받아 합산. 입력·출력·합계·비용을 한 줄로 노출.
- **최근 7일** — `start_time = now - 7d`, `bucket_width=1d`. 일별 7행 테이블 + 모델별 분해 카드.

시작 날짜가 `2026-01-01` 인 이유는 OpenAI Usage API 자체가 2024년 말 출시라 그 전 데이터는 없고, 실제 계정도 2026-01 이후 활동이 대부분이기 때문(운영자 메모 반영).

### 2. 엔드포인트 호출 구조

OpenAI SDK 에 Usage API 전용 메서드가 없으므로 **`requests.get` 직접 호출**.

```python
BASE = 'https://api.openai.com/v1/organization'
_HEADERS = {'Authorization': f'Bearer {admin_key}'}

params = {
    'start_time': int(start.timestamp()),
    'end_time':   int(end.timestamp()),
    'bucket_width': '1d',
    'limit': 180,            # 페이지네이션 여유
    'group_by[]': 'model',   # 모델별 분해용 (7일만)
}
r = requests.get(f'{BASE}/usage/completions', headers=_HEADERS, params=params, timeout=10)
```

- `/usage/completions` + `/usage/embeddings` → 토큰 집계
- `/costs` → 실제 청구 금액(통화 단위 USD)

페이지네이션은 `has_more` + `next_page` 를 순회(루프 3회 안전판). 현재 규모(최근 7일 기준)에선 한 페이지로 끝난다.

### 3. 응답 구조 가공

서비스에서 BO 엔드포인트로 넘길 최종 JSON (모달 친화 구조):

```json
{
  "total": {
    "input_tokens": 12300000,
    "output_tokens": 2100000,
    "total_tokens": 14400000,
    "cost_usd": 23.47
  },
  "last_7d": {
    "input_tokens": 850000,
    "output_tokens": 180000,
    "total_tokens": 1030000,
    "cost_usd": 1.42,
    "daily": [
      {"date": "2026-04-18", "input": 120000, "output": 30000, "total": 150000, "cost_usd": 0.21},
      ...
    ],
    "by_model": [
      {"model": "gpt-4o-mini",             "tokens": 950000, "cost_usd": 1.30},
      {"model": "text-embedding-3-small",  "tokens": 80000,  "cost_usd": 0.12}
    ]
  }
}
```

모달 JS 는 이 구조를 그대로 받아 렌더하고 계산을 더 하지 않는다 (서비스 레이어 계산 책임).

### 4. 에러 처리

| 상황 | BO 엔드포인트 응답 | 모달 표시 |
|---|---|---|
| `OPENAI_ADMIN_KEY` 미설정 | `503 {error: "admin_key_missing"}` | "Admin 키가 설정되지 않았습니다. `.env` 의 `OPENAI_ADMIN_KEY` 를 확인하세요." |
| OpenAI `401 Unauthorized` | `502 {error: "unauthorized"}` | "Admin 키 인증이 거부되었습니다. 키가 유효한지 확인하세요." |
| 네트워크 타임아웃 / 5xx | `502 {error: "upstream_failed"}` | "OpenAI 응답을 받지 못했습니다. 잠시 후 다시 시도해 주세요." |
| 예상치 못한 포맷 | `500` (로그) | "데이터를 해석하지 못했습니다." |

모달 안에 **재시도 버튼** 하나만 둔다 — 다시 fetch 후 렌더.

### 5. 모달 컴포넌트

`DesignGuideline` 의 `.modal-overlay / .modal / .modal-header / .modal-title / .modal-close / .modal-body / .modal-footer` 그대로 사용. bo.css 에 해당 섹션이 없으므로 그대로 포트.

- 너비는 기본 `480px` 에서 본 위젯이 카드·표가 여러 개라 **`720px` 로 오버라이드** (`.modal.modal-wide { width: 720px; }`).
- 닫기 동작: × 버튼 / 백드롭 클릭 / ESC 키 세 가지.
- 스크롤: `max-height: 80vh; overflow: auto` 로 본문이 길어지면 본문만 스크롤.

### 6. 버튼 배치

```html
<header class="page-header">
  <div>
    <h1 class="page-title">대시보드</h1>
    <p class="page-description">최근 {{ days }}일 동안의 AI 토큰 사용량</p>
  </div>
  <button type="button" class="btn btn-primary" id="openai-usage-open">API 사용량</button>
</header>
```

Phase 4-2 의 `+ 새 규칙` 버튼과 같은 자리·톤. 라벨만 "API 사용량".

### 7. 테스트 전략

OpenAI 실제 호출 없이 `requests.get` 을 mock. 4 케이스:

- `test_missing_admin_key_raises_controlled_error` — `OPENAI_ADMIN_KEY=""` 이면 `AdminKeyMissing` 예외.
- `test_unauthorized_propagates_as_upstream_error` — 401 응답 → 커스텀 `UsageAPIError`.
- `test_aggregates_last_7d_daily_buckets` — mock 응답을 총합·일별·모델별로 잘 묶는지.
- `test_handles_empty_result_list` — 버킷은 있는데 `results=[]` 인 날 (호출 없음) 0 으로 집계.

## 리스크 & 대응

| 리스크 | 대응 |
|---|---|
| Admin 키 유출 시 조직 전체 노출 | `.env` 만 사용, BO 응답에는 키 미노출, 로그에도 출력 금지. 필요 시 `gitignore` 재확인 |
