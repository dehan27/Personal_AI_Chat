# 2.0.0 Phase 1 — Prompt 관리 분리 개발 플랜

## Context

현재 프롬프트는 `chat/prompt/chat.py`의 `SYSTEM_PROMPT`와 `chat/services/prompt_builder.py` 내부의 3개 상수(`_SOURCES_INSTRUCTION`, `_QA_INSTRUCTION`, `_NO_SOURCES_GUARD`)로 코드에 직접 박혀 있다. `history_service.py`에서도 `SYSTEM_PROMPT`를 import해서 쓴다. 이 구조는 문구 한 줄을 바꾸려 해도 재배포가 필요하고, 2.0.0 로드맵의 LangGraph 도입 시 node별 프롬프트 분리도 어렵다.

Phase 1의 목표는 "파일 기반 Prompt 관리 + Loader 계층 + BO 편집 UI" 인프라를 심어두는 것. 이후 Phase(LangGraph 진입, router/workflow/agent 프롬프트 추가)가 이 로더를 그대로 재사용할 수 있도록, **인터페이스는 단순하되 확장 여지를 남긴다**.

이 Phase는 기능 동작에는 변화가 없다. `single-shot` 답변이 그대로 유지되어야 하고, 단지 프롬프트 출처만 `.md` 파일로 이동한다.

## Scope

**포함**
- 시스템 프롬프트 + single-shot 보조 지시문 4종 파일 분리
- `PROMPTS_DIR` 설정 (기본값 + 환경변수 override)
- `load_prompt()` 로더 (파일 누락 시 예외, 얇은 in-process 캐시)
- BO Prompt 관리 페이지 (목록 + 별도 편집 페이지)
- Registry 기반 allow-list 방식의 안전 저장
- 사이드바 메뉴 + DesignGuideline 가이드라인 준수
- README 갱신

**제외 (이후 Phase로)**
- DB 기반 프롬프트 저장
- 버전 diff·rollback UI
- 환경 분기(dev/staging/prod) 프롬프트
- draft/publish 승인 플로우
- LangSmith 연동
- router/workflow/agent용 프롬프트 실사용 (디렉토리 구조만 준비)

### Step 3 — Prompt Loader 구현
**신규 파일**: `chat/services/prompt_loader.py`

**API**
```python
from pathlib import Path
from django.conf import settings

class PromptNotFound(Exception):
    """프롬프트 파일이 존재하지 않음. 배포 누락을 조기에 드러내기 위해 예외를 던진다."""

def load_prompt(relative_path: str) -> str:
    """PROMPTS_DIR 하위의 프롬프트 파일을 읽어 반환. 파일 없으면 PromptNotFound."""

def save_prompt(relative_path: str, content: str) -> None:
    """프롬프트 파일을 저장. path traversal 차단을 위해 PROMPTS_DIR 내부인지 검증."""
```

**캐시 정책**: Phase 1은 모듈 레벨 dict로 얕게. 저장(`save_prompt`) 시점에 해당 키 invalidate.

**에러 정책**: 파일 누락 → `PromptNotFound` 예외. 읽기 실패는 조용히 삼키지 않음.

**보안 정책**: `save_prompt`는 `(PROMPTS_DIR / relative_path).resolve()`가 `PROMPTS_DIR.resolve()` 하위인지 확인. 아니면 `ValueError`.

**커밋**: `feat: Add prompt_loader with file-based load/save`

### Step 4 — Prompt Registry 정의
**신규 파일**: `chat/services/prompt_registry.py`

```python
from dataclasses import dataclass

@dataclass(frozen=True)
class PromptEntry:
    key: str               # URL 슬러그로 사용. 소문자+하이픈
    title: str             # BO 화면 표시명
    description: str       # 역할·주의사항
    relative_path: str     # PROMPTS_DIR 기준 상대경로
    editable: bool = True

PROMPT_REGISTRY: list[PromptEntry] = [
    PromptEntry('chat-system', '채팅 시스템 프롬프트', '답변 톤·응답 원칙', 'chat/system.md'),
    PromptEntry('chat-source-instruction', '회사 자료 답변 지시문', '...', 'chat/source_instruction.md'),
    PromptEntry('chat-qa-instruction', '과거 참고 답변 지시문', '...', 'chat/qa_instruction.md'),
    PromptEntry('chat-no-sources-guard', '자료 없음 가드', '...', 'chat/no_sources_guard.md'),
]

def get_entry(key: str) -> PromptEntry | None: ...
def all_entries() -> list[PromptEntry]: ...
```

**원칙**: BO는 오직 이 registry를 통해서만 파일에 접근. 사용자가 URL에 임의 경로 넣어도 allow-list 밖이면 404.

**커밋**: `feat: Define prompt registry as allow-list for BO editor`
