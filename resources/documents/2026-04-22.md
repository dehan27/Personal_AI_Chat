# 2026-04-22 개발 로그

## 1. XLS/XLSX 지원 추가 (PR #2)

**배경**
기존 업로드 파이프라인은 `.txt / .md / .pdf / .docx`만 허용. 개인 자료 중 규정 별첨·권한표·결재표 등이 엑셀로 관리되는 경우가 많아, 엑셀도 1급 시민으로 지원 필요.

**파서 추가 (`files/services/extractor.py`)**
- `_extract_xlsx` — `openpyxl` 사용, `data_only=True`로 **수식 캐시값** 읽기
- `_extract_xls` — `xlrd`로 레거시 포맷 지원 (날짜·불리언·빈 셀 타입별 파싱)
- **병합 셀 복제**: 좌상단 값만 있는 병합 범위를 (row, col) 맵으로 전개해 전체 셀에 값 복제 → 헤더 병합 인식 정확도 향상
- **셀 값 정규화** (`_format_cell_value`): `datetime` → ISO 문자열, 정수형 `float` (예: 5.0) → 정수 문자열, 공백 trim
- 시트당 **10,000행 soft limit** (`XLSX_MAX_ROWS_PER_SHEET`) + 절단 시 경고 로그
- `sheet_state != 'visible'`인 숨김 시트는 스킵

**업로드 허용 확장자 갱신 (`bo/views/files.py`, `bo/templates/bo/files.html`)**
- `ALLOWED_EXTS`에 `.xlsx / .xlsm / .xls` 추가
- HTML `<input accept="{{ allowed_exts }}">`로 템플릿 변수화 (이전에는 하드코딩이라 서버/클라이언트 불일치로 엑셀이 파일 선택창에서 걸러지던 버그 수정)

**의존성 추가 (`requirements.txt`)**
- `openpyxl`, `xlrd` (xlrd는 `.xls` 전용; `.xlsx`는 보안상 미지원 버전 사용)

## 2. XLSX 와이드 희소 매트릭스 추출 개선

**배경**
결재 라인표·권한 매트릭스처럼 열이 많고 대부분 빈 셀인 엑셀 표를 기존 마크다운 파이프 테이블로 변환하면, LLM이 어느 열이 어느 헤더에 대응하는지 잃어버려 RAG 응답 품질이 떨어짐. 특정 파일 전용이 아닌 **일반 휴리스틱**으로 해결.

**감지 규칙 (`files/services/extractor.py`)**
- 열 수 ≥ `MATRIX_MIN_COLS` (10) **그리고** 데이터 행의 빈 셀 비율 ≥ `MATRIX_EMPTY_RATIO` (0.6) 이면 매트릭스로 간주
- 일반 조밀 표는 기존 마크다운 표 유지

**헤더 블록 탐지**
- `_detect_header_block`: 채움 비율 ≥ 70%인 첫 행을 `start`, 아래로 ≥ 50%인 행까지 `end` 확장 (다단 헤더 대응)
- 헤더 이전 행들은 `prelude`(타이틀/설명)로 분리되어 본문 앞에 출력
- 헤더 블록 내 같은 열의 값들을 중복 제거 후 결합하여 최종 컬럼 헤더 구성

**출력 형식**
```
[별첨] 위임전결규정 - 결재 라인

[행 1] 구분: 인사 | 항목: 1) 근태신청 (외출, 조퇴) | 서류: 근태신청/사유서 | 기안자: O | 파트장: O | PM: V
[행 2] 항목: 2) 휴가 신청 | 서류: 휴가신청서 | 기안자: O | 파트장: O | 팀장: V
```

**검증 케이스**
| 케이스 | 조건 | 동작 |
| --- | --- | --- |
| 23열 결재표 (단일 헤더) | 매트릭스 | key-value 분기 |
| 14열 권한 매트릭스 (다단 헤더) | 매트릭스 | key-value 분기, 다단 헤더 결합 |
| 5열 직원 목록 (조밀) | 일반 표 | 기존 마크다운 유지 |

`.xlsx` / `.xlsm` / `.xls` 공통으로 `_rows_to_text()` 통과. pdf/docx 표는 영향 없음.

## 3. Q&A 관리 — 대화로그 변경

**탭 순서 변경 (`bo/templates/bo/qa_logs.html`)**
- 기존: `미승격 / 승격됨 / 전체`
- 변경: `전체 / 미승격 / 승격`
- `bo/views/qa.py` `qa_logs` 기본 탭도 `'pending'` → `'all'`

**공식 Q&A 승격 ChatLog 삭제 제한**
- 요구사항: 공식 Q&A 창이 아닌 곳에서는 승격된 ChatLog 삭제 불가
- UI: `qa_logs.html`, `qa_feedback.html`에서 `promoted_count > 0`이면 삭제 버튼 숨김
- 서버 가드: `qa_log_delete`에서 `promoted_count` 확인 → 승격 상태면 에러 메시지 반환 + 원 페이지로 리다이렉트
- 승격된 ChatLog를 삭제하려면 공식 Q&A 창에서 CanonicalQA를 먼저 정리해야 함

## 4. 채팅 UI — 최신 메시지 자동 스크롤 수정

**문제**
기존 `chat.js`의 `scrollToBottom()`이 `#messages`(= `.chat-frame`, overflow 없음)에 `scrollTop`을 설정. 실제 스크롤 컨테이너는 부모 `.chat-messages`(`<main>`)이라 전혀 움직이지 않음.

**수정 (`chat/static/chat/chat.js`)**
- 스크롤 대상을 `document.querySelector('.chat-messages')`로 변경
- `requestAnimationFrame`으로 감싸 메시지 append → 레이아웃 반영 → 다음 프레임에 스크롤되도록 조정
